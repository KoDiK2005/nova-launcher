# NOVA Launcher — Шаг 1: offline-запуск игры

> Дата: 2026-06-18
> Тема: как лаунчер скачивает версию и стартует Minecraft без логина

## Что теперь умеет проект

Полный цикл offline: выбрал версию → нажал ИГРАТЬ → если версии нет, она
качается с полоской прогресса → запускается java с игрой. Без Microsoft-логина.

## Три кита логики (`backend/minecraft.py`)

1. **get_versions()** — спрашивает у серверов Mojang список версий, отдаёт релизы.
2. **install_version(id, callback)** — качает версию в `game_data/`. callback —
   словарь с `setStatus/setProgress/setMax`, либа дёргает их → рисуем полоску.
3. **launch_offline(id, username)** — собирает `options` и команду для java,
   запускает через `subprocess.Popen` (не блокирует лаунчер).

## offline-UUID — почему не случайный

```python
data = hashlib.md5(f"OfflinePlayer:{username}".encode()).digest()
uuid.UUID(bytes=data, version=3)
```
Minecraft в offline сам так делает: ник → всегда ОДИН и тот же uuid.
Поэтому миры/инвентарь/скин привязаны к нику стабильно между запусками.
Если б делали случайный uuid4 — каждый запуск был бы "новым игроком".

## Почему play() работает в отдельном потоке (`backend/api.py`)

Скачивание версии — это минуты. Если делать это прямо в вызове из JS, окно
залипнет ("Не отвечает"). Поэтому `play()` стартует `threading.Thread` и сразу
возвращает управление, а прогресс шлёт обратно во фронт через
`window.evaluate_js("onProgress(...)")`. Морда живёт, полоска ползёт.

```
JS: play()  ──>  Api.play() запускает поток ──> сразу return {ok:true}
                         │
                         └─ поток: install (онпрогресс→JS) → launch → onLaunched→JS
```

## Грабли, на которые напоролись

- **PowerShell execution policy** — venv\Scripts\activate не запускался.
  Лечится: `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned`.
- **OneDrive + null bytes** — py_compile иногда ругается на нулевые байты,
  если OneDrive синхронит файл прямо в момент чтения. Ложняк, повторить.
- **Нужна Java 17+** в PATH, иначе игра не стартует.

## Новые слова

- **js_api / evaluate_js** — двусторонний мост: JS зовёт Python-методы,
  Python дёргает JS-функции.
- **callback (колбэк)** — функцию передают внутрь другой функции, чтобы та
  её вызывала в нужный момент (тут — на каждый шаг загрузки).
- **поток (thread)** — параллельная дорожка выполнения, чтобы долгая задача
  не морозила интерфейс.
