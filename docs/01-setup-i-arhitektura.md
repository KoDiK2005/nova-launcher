# NOVA Launcher — Шаг 0: Архитектура и каркас

> Дата: 2026-06-18
> Тема: с чего вообще начинается десктопный лаунчер

## Стек и почему именно он

- **Python** — язык логики (мой стек, знаю).
- **pywebview** — оборачивает HTML/CSS-макет в десктопное окно. UI не переписываем.
- **minecraft-launcher-lib** — готовая либа: качает версии, собирает команду запуска java, умеет offline и Microsoft-вход.

Альтернативы и почему мимо: Electron (утащил бы в JS), Go+Wails / Tauri (новый язык в стек ради ничего на старте).

## Главный принцип: разделение ответственности

Три слоя, каждый знает только своё:

| Слой | Файл | Отвечает за |
|------|------|-------------|
| UI | `NOVA Launcher.html` | как выглядит, кнопки |
| Контроллер / мост | `backend/api.py` | принять вызов из JS, дёрнуть логику |
| Логика | `backend/minecraft.py` | скачать/запустить игру |

Зачем так: сломается морда — логика жива. Меняем либу — UI не трогаем.
Это не «для красоты», это чтобы проект не превратился в один файл на 2000 строк, в котором сам чёрт ногу сломит.

## Как JS зовёт Python

В `main.py` передаём `js_api=Api()` в окно. После этого во фронте:

```js
const versions = await window.pywebview.api.list_versions();
await window.pywebview.api.play("1.20.4", "Mark");
```

Каждый публичный метод класса `Api` автоматически виден из JS и возвращает Promise.

## Что дальше (мой кусок кода)

Дописать `launch_offline()` в `minecraft.py`:
1. собрать `options` (username, uuid, token="")
2. `mll.command.get_minecraft_command(...)`
3. `subprocess.Popen(command)`

## Новые слова, которые надо запомнить

- **offline-режим** — запуск по нику без проверки лицензии Microsoft. token пустой.
- **subprocess.Popen** — запускает внешнюю программу (java) в отдельном процессе, НЕ блокируя наш лаунчер.
- **callback прогресса** — функция, которую либа дёргает во время скачивания, чтобы мы рисовали полоску загрузки.
