# NOVA Launcher

Десктопный лаунчер Minecraft с кастомным UI. Без Microsoft аккаунта, с поддержкой модов и мультиплеером напрямую между друзьями.

## Возможности

### Игра
- Запуск любой версии Minecraft (release) в offline-режиме
- Поддержка Fabric мод-лоадера (версии 1.14+)
- Выбор объёма RAM (512 MB — 8 GB)
- Настройка разрешения окна игры
- Кастомный путь к Java и дополнительные JVM аргументы
- Ник и настройки сохраняются между сессиями
- Стабильный UUID на основе ника (инвентарь не теряется)

### Мультиплеер (без Microsoft, без Radmin/Hamachi)
- Встроенный Minecraft сервер — запуск одной кнопкой
- **Собственный P2P-туннель** — прямое подключение к другу без ручного проброса порта
  - NAT traversal через STUN + UDP hole punching (работает через большинство домашних роутеров)
  - Шифрование туннеля (Curve25519 + XSalsa20-Poly1305, PyNaCl) — трафик нельзя перехватить или подменить
  - Авторизация только по списку друзей лаунчера + закрепление ключа друга при первом подключении (TOFU)
  - Keep-alive и автопереподключение — короткие потери пакетов не выкидывают из игры
  - Фолбэк на TCP-relay, если оба игрока за Symmetric NAT (см. [docs/P2P.md](docs/P2P.md))
- Автоматический проброс порта через **UPnP** остаётся как запасной вариант (`upnp.py`)
- Онлайн список друзей — видно кто сейчас хостит и на какой версии
- Кнопка **▶ ИГРАТЬ (P2P)** прямо в карточке друга
- `online-mode=false` выставляется автоматически — аккаунт не нужен

### Интерфейс
- Три вкладки: Главная / Мультиплеер / Настройки
- Список серверов с быстрым подключением
- Кнопка открытия папки модов и папки игры

## Зависимости

- Python 3.12+
- Java 17+ в PATH (`java -version`)

## Установка

```powershell
git clone https://github.com/KoDiK2005/nova-launcher.git
cd nova-launcher
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

## Структура

```
launcher/
├── main.py                — точка входа
├── requirements.txt
├── config.json            — настройки пользователя (gitignore)
├── backend/
│   ├── api.py             — мост JS ↔ Python, все методы API
│   ├── minecraft.py       — загрузка, Fabric, запуск игры
│   ├── server_manager.py  — встроенный MC сервер
│   ├── upnp.py            — UPnP проброс порта (запасной путь)
│   ├── relay.py           — MQTT: presence + обнаружение друзей онлайн
│   └── p2p/               — NAT traversal и шифрованный туннель (замена Hamachi/Radmin)
│       ├── stun.py            — STUN-клиент, определение публичного IP и типа NAT
│       ├── crypto_box.py      — identity игрока (Curve25519) + шифрование туннеля
│       ├── punch.py           — UDP hole punching
│       ├── signaling.py       — обмен offer/answer между друзьями через MQTT
│       ├── reliable_udp.py    — надёжный канал поверх UDP (ack/retransmit/keep-alive)
│       ├── turn_relay.py      — TCP relay-фолбэк для Symmetric NAT
│       ├── tunnel.py          — мост Minecraft(TCP, localhost) ↔ P2P-туннель
│       └── manager.py         — публичное API подсистемы (используется из api.py)
├── ui/
│   └── index.html         — весь интерфейс (3 страницы)
└── server_data/           — файлы серверов (gitignore)
```

## Как играть с друзьями

1. **Хозяин**: вкладка **Мультиплеер** → выбрать версию → **▶ ХОСТИТЬ**.
   Лаунчер поднимает встроенный сервер локально (`online-mode=false`, аккаунт не нужен)
   и публикует свой статус "хостит" друзьям через сигнальный сервер.
2. **Хозяин и друг должны быть друг у друга в списке друзей** (вкладка **Мультиплеер** →
   поле «Добавить друга» по нику) — P2P-сессию может инициировать только тот,
   кто есть в списке, это и есть авторизация.
3. **Друг** видит у хозяина в карточке статус **🚀 хостит** и кнопку
   **▶ ИГРАТЬ (P2P)** — жмёт её.
4. Дальше всё происходит автоматически, без проброса портов и сторонних программ:
   - лаунчеры обоих игроков обмениваются кандидатами через MQTT (STUN + локальный адрес);
   - происходит UDP hole punching — пробивается NAT в обе стороны;
   - поднимается зашифрованный туннель (если punching не удался — теряются и UDP, и
     прямой путь — используется TCP relay-фолбэк, если он настроен; иначе показывается
     ошибка "оба за Symmetric NAT");
   - Minecraft у друга подключается к `127.0.0.1:<локальный порт>` — для игры это
     обычный сервер, IP хозяина никогда не вводится руками.
5. Если соединение коротко обрывается (провайдер/Wi-Fi) — лаунчер сам переподключается,
   игрока не выкидывает с сервера.

Подробное описание протокола и архитектуры — в [docs/P2P.md](docs/P2P.md).

Если P2P недоступен (оба за Symmetric NAT и relay не настроен) — можно по-старому
открыть порт через **UPnP** в настройках хоста и передать другу IP вручную.
